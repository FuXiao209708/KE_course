# 班级学情分析页面说明

## 功能概述

班级学情分析页面是知识工程课程智能伴学平台的第五个核心功能模块，为助教和教师提供全方位的班级学习状态洞察。

## 功能模块

### 1. 核心学情大盘（KPI 卡片）
位于页面顶部，通过四个精美的数据卡片展示关键指标：

- **活跃度**：周均活跃率 87%，反映学生使用平台的频率
- **互动频次**：人均提问 12.3 次/周，衡量学生与AI助手的互动程度
- **作业进度**：当前批次提交率 78%，监控作业完成情况
- **答疑效率**：平均响应时长 2.4 小时，评估论坛答疑速度

### 2. 知识图谱"盲区"热力图
这是最具价值的特色模块，结合课程知识图谱展示学生的知识掌握情况：

#### 知识盲区排行（左侧）
- 基于习题智能评判的反馈数据
- 统计得分率最低的知识点
- 显示错误人数和平均得分
- 采用渐变色柱状图可视化
- Top 5 盲区：
  1. SPARQL属性路径（52分，23人错误）
  2. OWL约束推理（58分，19人错误）
  3. 本体对齐算法（61分，17人错误）
  4. TransE损失函数（65分，15人错误）
  5. 知识融合策略（68分，14人错误）

#### 高频检索知识点 Top 5（右侧）
- 展示图谱节点被查看次数最多的知识点
- 反映学生主动学习的热点话题
- 帮助教师了解学生关注的重点
- 包含知识点名称、所属模块和查看次数

### 3. 学习投入度 vs 学习效果（双轴趋势图）
采用柱状图+折线图的组合形式：

- **横轴**：教学周（第1周 - 第7周）
- **左纵轴（柱状图）**：周均使用时长（小时）
- **右纵轴（折线图）**：班级平均分
- **价值**：验证"多用平台/多和AI交流"是否能正向提升成绩
- **趋势**：从第1周的3.2小时/65分，提升到第7周的12.3小时/85分

### 4. 错题原因维度分析（堆叠柱状图）
将习题错误归类为四个维度：

1. **概念理解模糊**（38%，45次）：如区分不清RDF和OWL
2. **SPARQL语法错误**（27%，32次）：查询语句编写错误
3. **算法计算错误**（23%，28次）：如Levenshtein距离算错
4. **逻辑推理不足**（12%，14次）：推理过程不完整

帮助识别学生是卡在理论理解还是工程实践上。

### 5. 学生提问意图分类（环形进度图）
对AI助手交互文本进行意图分析：

- **概念解释**（40%）：询问知识点定义和原理
- **代码Debug**（35%）：SPARQL查询、算法实现等调试
- **作业确认**（15%）：确认作业要求和提交方式
- **扩展延伸**（10%）：深入探讨和拓展学习

### 6. 论坛高频关键词（词云）
提取近一周论坛讨论的热点话题：

- 采用不同字号和颜色展示词频
- 高频词包括：SPARQL、FILTER、OPTIONAL、知识抽取、margin、实体对齐、TransE、OWL、本体推理、DBpedia、RDF、三元组
- 直观反映当下最火热的讨论话题

## 技术实现

### 前端技术栈
- **HTML5 + Tailwind CSS**：响应式布局
- **原生JavaScript**：无框架依赖，轻量高效
- **SVG图表**：自定义绘制趋势图和可视化组件
- **CSS动画**：平滑的过渡效果和交互反馈

### 数据可视化
- 柱状图：知识盲区排行、错题原因分析
- 折线图：学习趋势双轴图
- 环形进度图：提问意图分布
- 词云：论坛热点关键词
- 进度条：高频检索知识点

### 样式设计
- 与整体系统风格保持一致
- 采用深绿色（#065f46）和琥珀金（#d97706）主题色
- 卡片式布局，圆角阴影设计
- 响应式网格系统，支持移动端

## 使用方法

### 启动服务器
```bash
cd "f:\SEU_projects\2605知识工程课视频\demo"
python -m uvicorn backend.app:app --host 127.0.0.1 --port 8000
```

## 数据说明

当前使用的是**模拟数据**（Mock Data），用于演示展示。数据包括：

- 7周的学习趋势数据
- 5个知识盲区和5个高频检索点
- 4类错题原因统计
- 4种提问意图分布
- 12个论坛高频关键词

在实际部署时，可以通过后端API接入真实数据源。

## 设计亮点

1. **数据驱动决策**：通过可视化帮助教师快速定位教学重点
2. **多维度分析**：从活跃度、掌握度、错误类型等多角度评估
3. **知识图谱联动**：结合课程知识图谱，精准定位知识盲区
4. **趋势预测**：通过历史数据展示学习效果的演进
5. **交互友好**：悬停效果、动画过渡，提升用户体验

## 教学价值

- **助教视角**：一眼看出下周上课需要重点串讲哪个知识点
- **教师视角**：了解整体教学效果，调整教学策略
- **学生视角**：（可扩展）个人学情报告，了解自己的薄弱环节
- **教学闭环**：验证平台使用与成绩提升的正向关系
